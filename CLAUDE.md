# Claude Code project instructions

## Rules

- Work conservatively.
- Prefer small, reviewable changes.
- Do not expose secrets, tokens, keys, credentials, or private environment values.
- Do not rewrite unrelated files.
- Inspect the repository structure before making changes.
- Prefer pull requests for code changes.

## Verification

- Run available tests, linters, or build commands when discoverable.
- If tests are unavailable, explain what was checked manually.
