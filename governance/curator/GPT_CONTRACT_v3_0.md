## ⚡ ГЛАВНЫЙ МЕХАНИЗМ: САМОПОДДЕРЖАНИЕ ЧАТА

**Чат живёт пока GPT сам себе ставит следующую задачу.**

Алгоритм после каждой задачи:
```
1. Выполнил задачу → обновил ACTIVE_QUEUE (DONE)
2. Написал отчёт оператору
3. СРАЗУ написал себе: "Следующая задача: <id> — приступаю"
4. Выполнил следующую задачу
5. Повторить с шага 1
```

**Чат НЕ прекращается пока GPT не напишет оператору "ACTIVE_QUEUE пуст".**
Ожидание = смерть чата = нарушение контракта.

## A5. АВТОРЕМОНТ — ОБЯЗАТЕЛЕН

При любой ошибке — НЕ останавливаться. НЕ ждать оператора.

| Ошибка | Алгоритм | Запрещено |
|---|---|---|
| Workflow упал | `get_file_contents` → упавший yml/py → найти баг → исправить → `create_workflow_dispatch` | Перезапускать не читая |
| Receipt отсутствует | `get_file_contents → governance/proofs/` → если нет файла → workflow упал → авторемонт | Считать "не могу проверить" |
| SyntaxError в .py | Прочитать файл → исправить опечатки/отступы → записать с sha | Записать с ошибкой |
| sha=null | Повторить `create_or_update_file` немедленно | Считать DONE |
| ACTIVE_QUEUE пуст | Сообщить оператору → ждать | Придумывать задачи |
| Все задачи BLOCKED | Починить первый технический блокер автономно | Останавливаться |

## A6. КАК ПРОВЕРИТЬ RECEIPT САМОСТОЯТЕЛЬНО

GPT **может и обязан** проверять receipts сам:

```
get_file_contents → governance/proofs/BEM931-V36-RM15_live_e2e_receipt.json

Если файл существует → прочитать status внутри
Если файл отсутствует (404) → workflow упал → авторемонт
```

**"Я не могу проверить результат workflow"** = нарушение контракта.
get_file_contents работает всегда. Receipts пишутся в репо. Репо читается через MCP.

## A7. ОБЯЗАТЕЛЬНАЯ ПРОВЕРКА .PY ПЕРЕД ЗАПИСЬЮ

| # | Проверка | Типичная ошибка GPT |
|---|---|---|
| 1 | Нет опечаток в строках | `replace('-', '_'i)` — лишняя `i` внутри строки |
| 2 | Правильные отступы (4 пробела) | 3 пробела в if/else |
| 3 | Все импорты есть в bem931_runner_lib.py | import несуществующего имени |
| 4 | def main() присутствует | Файл без точки входа |
| 5 | if __name__ == '__main__': в конце | Файл не запускается standalone |
| 6 | channel_name формула | `contour_id.lower().replace('-','_') + '_to_' + suffix + '.jsonl'` |
| 7 | В YAML: нет двойных двоеточий в ключах | `"created_at"":` → `"created_at":` |
| 8 | В YAML: glob расширение | `*.jsol` → `*.jsonl` |

## A11. ПРИ ИСЧЕРПАНИИ ЛИМИТОВ ЧАТА
