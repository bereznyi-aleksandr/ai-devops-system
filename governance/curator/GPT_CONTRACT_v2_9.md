## A4. АВТОНОМНЫЙ SCHEDULER — ОБЯЗАТЕЛЬНО

GPT обязан поддерживать непрерывное выполнение. Механизмы:

| Механизм | Как использовать |
|---|---|
| GitHub Actions schedule | `cron: '*/5 * * * *'` — каждые 5 минут |
| repair-gate workflow | `bem931-v36-release-repair-gate.yml` — перезапускать при ошибке |
| Немедленный retry | sha=null или ошибка 4xx → повторить в той же сессии |

**Scheduler НЕ является основанием для остановки. Запустил scheduler → продолжил следующую задачу.**

---

## A5. АВТОРЕМОНТ — ОБЯЗАТЕЛЕН

При любой ошибке — НЕ останавливаться. НЕ сообщать оператору до исправления.

| Ошибка | Алгоритм |
|---|---|
| Workflow упал | Читать runner.py → найти ошибку → исправить → перезапустить |
| Receipt отсутствует | Проверить governance/proofs/ → перезапустить workflow |
| SyntaxError в .py | Исправить опечатки/отступы → записать с sha |
| sha=null | Повторить create_or_update_file немедленно |
| ACTIVE_QUEUE пуст | Сообщить оператору и ждать |
| Все задачи BLOCKED | Починить первый технический блокер автономно |

## A6. ОБЯЗАТЕЛЬНАЯ ПРОВЕРКА .PY ПЕРЕД ЗАПИСЬЮ

| # | Проверка | Типичная ошибка |
|---|---|---|
| 1 | Нет опечаток в строках | `replace('-', '_'i)` — лишняя `i` |
| 2 | Правильные отступы | 3 пробела вместо 4 |
| 3 | Все импорты есть в bem931_runner_lib.py | from lib import NONEXISTENT |
| 4 | def main() присутствует | Файл без точки входа |
| 5 | if __name__ == '__main__': в конце | Файл не запускается standalone |
| 6 | channel_name: `contour_id.lower().replace('-','_') + '_to_' + suffix + '.jsonl'` | Неправильное имя канала |
