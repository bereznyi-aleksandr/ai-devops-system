**Этот файл больше не является источником истины.**

Контракт куратора v6.1 загружен напрямую в Knowledge-конфигурацию кастомного GPT (ChatGPT Custom GPT → Configure → Knowledge). Это единственный источник истины для контракта.

GPT читает контракт из своей конфигурации **гарантированно при каждом ответе** — это платформенная гарантия retrieval, не зависящая от вызова Action. Этот repo-файл GPT не обязан читать и не должен использовать как актуальную версию.

**Причина:** дублирование одного и того же контракта в двух местах (repo + Knowledge) создаёт риск расхождения версий — одна копия обновляется, другая нет, и непонятно какая актуальна.

**Разделение источников истины:**

| Что | Где живёт | Кто читает |
|---|---|---|
| Контракт (статичные правила, write-channel, deprecated-лист, compile-gate, evidence policy) | Knowledge-конфигурация GPT | GPT — гарантированно, каждый ответ |
| Живое состояние (текущая задача, прогресс roadmap, статусы провайдеров) | `governance/roadmap/ACTIVE_QUEUE.json`, `/AGENT_CONTEXT.md`, `governance/config/provider_config.json` | GPT — через read-Action, по необходимости |

Если нужно обновить контракт — обнови файл в Knowledge-конфигурации GPT напрямую. Этот файл (если вообще оставлен в репозитории) — только для человеческого чтения/истории, не для GPT.
