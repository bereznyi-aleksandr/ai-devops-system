---

## ЧАСТЬ 2. АВТОНОМНОСТЬ И НЕПРЕРЫВНОСТЬ

**Ключевое правило:** отчёт НЕ останавливает выполнение дорожной карты.

Цикл работы:
```
Выполнил шаг roadmap
→ записал результат в state
→ отправил отчёт оператору (параллельно, не вместо следующего шага)
→ немедленно перешёл к следующему шагу
→ цикл не останавливается никогда
```

Оператор подключается только если:
- GPT зафиксировал blocker который не может снять автономно
- Оператор явно обращается с запросом

## ЧАСТЬ 4. ВНУТРЕННИЙ КОНТУР РАЗРАБОТКИ

Целевая модель:
```
curator intake
  → analyst (анализ задачи, предложение решения)
  → auditor (проверка анализа и patch)
  → executor (применение через GitHub Actions + Python executor v3)
  → curator closure (закрытие цикла, отчёт)
```

State файлы:
- `governance/state/role_cycle_state.json` — состояние текущего цикла
- `governance/state/gpt_dev_session.json` — GPT developer runner сессия
- `governance/state/roadmap_state.json` — дорожная карта задач

## ЧАСТЬ 7. WRITE-CHANNEL GPT

Разрешённый автономный write-channel:
```
GPT → Deno createCodexTask
  → GitHub Actions ubuntu-latest
  → Python executor v3 (run_script mode)
  → commit → status
```

Проверка результата:
```
GPT → Deno getCodexStatus → completed / failed / queued + commit SHA + blocker
```
