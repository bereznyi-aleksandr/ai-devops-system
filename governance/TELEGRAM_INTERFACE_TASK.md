# ЗАДАЧА: TELEGRAM КАК ЕДИНЫЙ ИНТЕРФЕЙС КУРАТОРА
Версия: v1.0 | Дата: 2026-05-02 | BEM-270

---

## К ЧЕМУ МЫ ПРИШЛИ

GPT Tasks не доставляют отчёты в чат — только email/push уведомления.
Это ограничение платформы OpenAI которое нельзя изменить промптом.

**Решение: Telegram как единый интерфейс.**

Оператор работает с системой полностью через Telegram:
- Получает автоматические отчёты от куратора
- Ставит задачи через Telegram
- Не заходит в GitHub, не заходит в чаты

---

## АРХИТЕКТУРА

```
GitHub Actions (cron каждый час)
  → читает exchange.jsonl + issue #31 + MASTER_PLAN.md
  → формирует аналитический отчёт куратора
  → отправляет в Telegram оператору

GitHub Actions (по событию — завершение роли)
  → отправляет краткое уведомление в Telegram

Оператор пишет команду в Telegram
  → Telegram бот передаёт в GitHub Actions
  → Actions выполняет команду (@analyst / @auditor / @executor)
```

---

## ЧТО НУЖНО ОТ ОПЕРАТОРА (одноразово)

Добавить два секрета в GitHub:

```
github.com/bereznyi-aleksandr/ai-devops-system
→ Settings → Secrets and variables → Actions → New repository secret

1. TELEGRAM_BOT_TOKEN = [токен бота]
2. TELEGRAM_CHAT_ID   = [chat_id оператора]
```

**Как узнать TELEGRAM_CHAT_ID:**
Написать боту @userinfobot в Telegram → он пришлёт число.

## ЧТО СОЗДАЁТСЯ ПОСЛЕ

### 1. Workflow отчёта куратора (cron)

`.github/workflows/curator-telegram-report.yml`

Каждый час:
- Читает состояние системы
- Анализирует прогресс и отклонения
- Отправляет в Telegram:

```
📊 КУРАТОР | HH:MM UA

Этап E3 — 25%
Дорожная карта: 2/4

За последний час:
✅ [что выполнено]
⚠️ [что не выполнено / отклонения]
❌ [сбои если есть]

Темп: [в норме / отстаём / опережаем]
Следующий отчёт: HH:MM UA
```

### 2. Webhook — команды из Telegram в систему

`.github/workflows/telegram-webhook.yml`

Оператор пишет в Telegram:
- `статус` → куратор присылает текущий отчёт
- `запусти аналитика` → система запускает @analyst
- `что в репозитории` → список последних изменений
- `стоп` → пауза автономной работы
