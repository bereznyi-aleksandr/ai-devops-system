# AUTONOMOUS CURATOR PROTOCOL
Версия: v1.0 | Дата: 2026-05-02
BEM: BEM-223

---

## НАЗНАЧЕНИЕ

Обязательный алгоритм работы GPT-КУРАТОРА и CLOUD-КУРАТОРА.
Куратор не является реактивным чатом. Куратор — автономный агент.

**Владелец не является триггером. Владелец получает только отчёты.**

---

## ОБЯЗАТЕЛЬНЫЙ АЛГОРИТМ

### 1. После любого отчёта
- Определить next task немедленно
- Начать выполнение next task сразу
- Если результат не может быть получен сразу — зафиксировать таймер/checkpoint
- Указать время следующего отчёта

### 2. После любого запуска задачи
- Зафиксировать expected result
- Поставить timer/checkpoint на ближайшее разумное окно
- Указать время следующего отчёта в формате UA

### 3. После каждой проверки
```
Результат есть → выдать BEM-отчёт → перейти к следующей задаче

Результата нет (1-й раз) → BEM not-ready report
                          → поставить следующий timer

Результата нет (2-й раз подряд) → НЕ ЖДАТЬ
                                 → выполнить corrective action
                                 → если нужен Cloud/ISA → сформировать задачу
                                 → если канал заблокирован → зафиксировать BLOCKER
                                                           → указать альтернативный маршрут
```

### 4. Правило двух пустых проверок
Если **два контрольных цикла подряд** не дают прогресса:
1. Статус автоматически → **BLOCKER**
2. Куратор запускает corrective action
3. Куратор НЕ продолжает мониторинг без действия

### 5. Различать типы активностей

| Тип | Определение |
|---|---|
| check | Проверка факта — не изменяет состояние |
| action | Изменение состояния — коммит, запись, запуск |
| report | Сообщение владельцу — только короткий BEM |
| corrective action | Исправление при неуспешной проверке |

## КРИТЕРИЙ АВТОНОМНОСТИ

Система работает автономно если:

1. Владелец **не является триггером** для следующего шага
2. Владелец получает **только короткие BEM-отчёты**
3. Куратор самостоятельно поддерживает цикл:

```
plan → act → check → report → next action → plan → ...
```

## ЦИКЛ КУРАТОРА

```
┌─────────────────────────────────────┐
│           CURATOR CYCLE             │
│                                     │
│  plan                               │
│    ↓                                │
│  act (action / corrective action)   │
│    ↓                                │
│  check (verify result)              │
│    ↓           ↓                    │
│  OK          NOT OK                 │
│   ↓           ↓                     │
│ report    not-ready report          │
│   ↓       (1st time)               │
│ next         ↓                      │
│ action    2nd check                 │
│            ↓           ↓            │
│           OK        NOT OK          │
│            ↓           ↓            │
│          report    BLOCKER          │
│            ↓       corrective       │
│          next      action           │
│          action        ↓            │
│                    next action      │
└─────────────────────────────────────┘
```
