# BEM-580 | Operator Decision Readable Comparison Table

Дата: 2026-05-17 | 21:59 (UTC+3)

## Решение

Operator decision в Telegram должен показывать не набор строк, а читаемую сравнительную таблицу.

## Правильный формат Telegram

1. Каноничная шапка: BEM, этап, дорожная карта.
2. Чек-лист подготовки решения.
3. Ясный вопрос: что именно должен решить оператор.
4. Таблица сравнения с короткими ячейками.
5. Расшифровка только если нужна.
6. Рекомендация аудиторов.
7. Как ответить.

## Таблица

Telegram не поддерживает настоящие HTML-таблицы. Поэтому используем mobile-readable comparison table:

```text
Таблица сравнения:
┌────┬──────────────┬────────────────────┐
│ №  │ Вариант      │ Отличие / обоснование │
├────┼──────────────┼────────────────────┤
│ 1  │ Подтвердить  │ Быстро; уже согласовано │
│ 2  │ Доработать   │ Если вопрос неполный │
│ 3  │ Свой вариант │ Если нужен иной путь │
└────┴──────────────┴────────────────────┘
```

## Ограничение

В таблице должны быть короткие формулировки. Длинные тексты режутся до понятной длины и при необходимости выводятся ниже в блоке `Детали`.
