# BEM-575 | Operator Decision Telegram Canon Table

Дата: 2026-05-17 | 21:44 (UTC+3)

## Решение оператора

Telegram-сообщение для решения оператора должно соответствовать канону отчёта и быть самодостаточным.

## Обязательный формат

```text
BEM-<N> | OPERATOR DECISION REQUIRED | YYYY-MM-DD | HH:MM (UTC+3)

Этап: X/Y (Z%)
Дорожная карта: X/Y (Z%)

Чек-лист:
✅ Claude позиция получена
✅ GPT позиция получена
✅ Вопрос сформулирован
⚠️ Требуется решение оператора

Вопрос:
<понятный вопрос>

Таблица решений:
1) Вариант: <вариант 1>
   Обоснование: <почему этот вариант возможен>

2) Вариант: <вариант 2>
   Обоснование: <почему этот вариант возможен>

3) Вариант: Свой вариант / доработать
   Обоснование: если варианты неполные или вопрос требует уточнения

Рекомендация аудиторов:
<рекомендованный вариант и почему>

Как ответить:
Напиши: 1, 2, 3, да/подтверждаю или свой вариант текстом.
```

## Правило
Markdown pipe-table в Telegram не использовать: на мобильном она нечитаема. Использовать читаемую вертикальную таблицу решений.
