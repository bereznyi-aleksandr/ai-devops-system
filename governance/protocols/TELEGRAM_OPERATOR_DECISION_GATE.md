# BEM-562 | Telegram Operator Decision Gate

Дата: 2026-05-17 | 19:22 (UTC+3)

## Решение оператора

| Наименование | Описание | Обоснование |
|---|---|---|
| Claude-GPT sync | Claude и GPT синхронизируются асинхронно через repo artifacts / audit mailbox | Оператор не должен быть передаточным звеном |
| Operator decision | Итог согласования обязательно отправляется оператору в Telegram | Финальное решение принимает оператор |
| Decision channel | Telegram является каналом уведомления и фиксации необходимости решения | Оператор подтвердил этот канал |
| No auto-implementation | Если вопрос требует стратегического решения, реализация не начинается до operator decision | Контроль стратегических решений остаётся у оператора |

## Обязательный Telegram-шаблон для согласованных вопросов

```text
BEM-<N> | OPERATOR DECISION REQUIRED | YYYY-MM-DD | HH:MM (UTC+3)

Этап: <X>/<Y> (<Z%>)
Дорожная карта: <X>/<Y> (<Z%>)

Чек-лист:
✅ Claude дал позицию
✅ GPT дал позицию
✅ Разногласия/правки зафиксированы
⚠️ Требуется решение оператора

Наименование | Описание | Обоснование
Вопрос | <что согласовано Claude+GPT> | <почему нужно решение>
Рекомендация | <вариант> | <обоснование>
Решение оператора | ожидается | без решения не выполнять стратегический шаг
```

## Статусы решения

| Статус | Значение |
|---|---|
| pending_operator_decision | отправлено в Telegram, ждём решения |
| approved_by_operator | оператор одобрил |
| rejected_by_operator | оператор отклонил |
| revise_requested | оператор попросил доработать |

## Правило
Технические routine-задачи продолжаются по правилу непрерывности. Стратегические решения после Claude-GPT согласования идут в Telegram и ждут оператора.
