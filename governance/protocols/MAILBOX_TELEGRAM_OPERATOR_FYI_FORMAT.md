# BEM-571 | Mailbox Telegram Operator FYI Format

Дата: 2026-05-17 | 21:29 (UTC+3)

## Решение

| Наименование | Описание | Обоснование |
|---|---|---|
| Mailbox адресат | Сообщение из `claude_to_gpt` адресовано GPT, не оператору | Mailbox является каналом Claude↔GPT |
| Telegram роль | Telegram показывает оператору уведомление/аудит-трейл | Оператор не должен быть relay |
| Формулировка | В Telegram обязательно писать `Действие оператора: не требуется`, если это не strategic decision gate | Чтобы оператор не думал, что должен пересылать сообщение |
| Исключение | Если требуется стратегическое решение, использовать `OPERATOR DECISION REQUIRED` | BEM-562 |

## Mobile notification pattern

```text
BEM-MAILBOX | УВЕДОМЛЕНИЕ ОПЕРАТОРУ | workflow_runtime

Этап: 1/1 (100%)
Дорожная карта: 1/1 (100%)

Чек-лист:
✅ Новое сообщение mailbox обнаружено
✅ Адресат: GPT
✅ Записано в repo mailbox
✅ Действие оператора не требуется

Сообщение:
От: Claude
Кому: GPT
Тема: ...
Файл: governance/audit_mailbox/...

Действие GPT:
Прочитать файл и ответить через mailbox.

Действие оператора:
Не требуется. Это уведомление о синхронизации Claude↔GPT.
```
